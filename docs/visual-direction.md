# Helion Forge — Visual Direction

## Core Read
Helion Forge should look like a serious build shop for the AI era — not a glossy “future of everything” startup and not a sterile enterprise vendor. The feel is practical, high-signal, and engineered.

The brand promise is simple:

**Where intelligence builds.**

That line should anchor the page visually and emotionally. It implies agency, construction, systems, and output. Everything on the page should support that idea.

## Brand Character
- Builder-first, not trend-first
- Precise, not flashy
- Warm industrial, not cold sci-fi
- Confident, not inflated
- Technical, but readable by operators and founders

## Visual Mood
Think:
- dark workshop
- terminal glow without looking like a hacker cliché
- machined surfaces, structure, grids, frames
- calm competence
- orange used as heat/energy/focus rather than decoration

Avoid:
- generic SaaS gradients
- cartoon AI imagery
- humanoid robots
- overly abstract “innovation” blobs
- excessive glassmorphism or neon cyberpunk styling

## Color Direction
Use a restrained palette. Most of the page should be near-black and warm gray, with orange reserved for emphasis and action.

### Primary Palette
- **Forge Black** — `#0E0E10`
  - Main page background
- **Coal** — `#17181B`
  - Surface panels, cards, section contrast
- **Warm Gray** — `#B6AEA3`
  - Secondary text, dividers, quiet UI details
- **Ash** — `#E7E1D8`
  - Primary light text on dark areas, high-legibility moments
- **Forge Orange** — `#F36A21`
  - CTAs, key accents, highlights, active states
- **Ember Orange** — `#FF8A3D`
  - Hover states, glow edge, small emphasis moments

### Suggested Usage Ratio
- 65% near-black backgrounds
- 20% dark neutral surfaces
- 10% warm light text / muted gray structure
- 5% orange emphasis

### Contrast Notes
- Keep orange sparse so it feels valuable
- Body copy should lean Ash / muted warm gray, not pure white
- Use pure white rarely, if at all

## Typography Direction
The typography should feel industrial and modern.

### Recommended Pairing
- **Headlines:** Space Grotesk, Sora, or Geist
- **Body/UI:** Inter, Geist, or Instrument Sans

### Typography Behavior
- Headlines: compact, assertive, slightly tight
- Body: clean, neutral, readable
- Small labels: uppercase sparingly for section markers or system tags
- Avoid overly playful rounded typefaces

### Tone in Copy
Copy should sound like people who ship systems for a living.

Good:
- direct
- specific
- operational
- outcome-focused

Bad:
- visionary fluff
- inflated claims
- “revolutionizing” language
- vague AI buzzword stacking

## Layout Direction
Landing page structure should feel modular and engineered.

### Structural Principles
- Strong grid
- Generous spacing
- Obvious hierarchy
- Section breaks through tone and surface changes, not gimmicks
- Cards and panels should feel like functional containers, not decorative tiles

### Hero Direction
Hero should do three things fast:
1. state the thesis
2. establish credibility
3. point to action

Suggested hero composition:
- Small eyebrow line above headline
- Main headline featuring or supporting the tagline
- Brief supporting paragraph in plain English
- Primary CTA in orange
- Secondary CTA in neutral outline or text style
- Background texture: subtle grid, schematic linework, or dim industrial glow

### Section Rhythm
Recommended sequence:
1. Hero
2. What Helion Forge builds / enables
3. Why it works
4. Selected capabilities or engagement model
5. CTA block

## Imagery Direction
The site should not depend on heavy illustration. Use minimal, high-discipline visual elements.

### Best-fit Imagery Styles
- subtle technical diagrams
- abstract system maps
- grid overlays
- cropped macro textures: metal, matte panels, heat, structure
- soft orange light bloom used very selectively

### What to Avoid
- stock team photos
- handshake/business photography
- obvious AI brain imagery
- floating 3D cubes unless they serve a systems metaphor
- crowded dashboards as decoration

## UI Styling Notes
- Corners: slightly rounded, not bubbly
- Borders: thin, low-contrast, used to define structure
- Shadows: minimal; rely more on contrast than blur
- Buttons: solid orange primary, quiet dark secondary
- Icons: simple line icons or no icons at all
- Motion: restrained, fast, purposeful

## Homepage Tone Reference
If the page works, a visitor should think:
- these people build real systems
- they understand operators, not just audiences
- this is high-end and serious without pretending to be luxury
- the product feels durable

## Starter Asset Plan
Keep the first pass lean. Do not overproduce assets before the page direction is validated.

### Priority 1 — Needed for v1 landing page
1. **Wordmark / logotype lockup**
   - Horizontal version
   - Simple monochrome version
   - Orange-accent version
2. **Color tokens**
   - Hex values for primary palette
   - light/dark text usage notes
3. **Type system definition**
   - headline font
   - body font
   - scale for H1/H2/body/small text
4. **Hero background treatment**
   - one subtle textured backdrop
   - ideally CSS-first or lightweight SVG
5. **Basic UI kit**
   - button styles
   - section spacing rules
   - card/panel treatment
   - border and radius rules

### Priority 2 — Helpful next
1. **Simple brand mark exploration**
   - abstract forge / frame / beam / spark concept
2. **Lightweight icon set direction**
   - stroke weight and style guidance
3. **Reusable background motifs**
   - grid pattern
   - schematic lines
   - noise/texture layer
4. **Social/share image template**
   - dark background, orange accent, tagline lockup

### Priority 3 — Only if needed later
1. Product or process diagrams
2. Case-study thumbnail system
3. Illustration system for features or workflows

## Suggested Initial Asset Formats
- **Logo:** SVG first, PNG exports second
- **Patterns/backgrounds:** SVG or CSS where possible
- **Social card:** PNG at standard social sizes
- **Icons:** SVG

## Practical Design Constraints
- Keep the page fast
- Prefer vector and CSS treatments over heavy raster art
- Make the page feel custom without needing a giant asset pipeline
- Every visual decision should support trust, clarity, and build credibility

## One-line Creative Direction
**A dark, engineered landing page with restrained heat — built for people who care more about systems shipping than startup theater.**
